JSP、EL与JSTL

JSP

该技术现已被Spring、Spring MVC、mybatis等技术取代，内容了解即可，便于后续框架学习的理解

  • Java Server Page，是Java的服务器页面，主要作用是像回传html页面的数据，解决回传凭借Java代码繁琐的问题

  • 创建JSP页面：右键web目录 ==》 新建JSP ==》 创建文件名xx.jsp

  • 访问地址：http://ip:port/工程路径/xx.jsp

jsp为什么被淘汰了？

  • jsp被淘汰本身并不是因为技术落后的原因，它之所以被淘汰是因为行业趋势。

  • 现在强调前后端分离，前端写前端的代码，后端写后端的代码，没有必要将前端和后端代码融合在一起，所以jsp使用就受限了。

  • 并不是完全就没人用了，只是用的少了

JSP本质

  • jsp页面本质上是Servlet程序
  • 第一次访问jsp页面，服务器会将其翻译成java源文件，并对它进行编译成一个字节码文件.class
  • 追入源代码发现xx.jsp类继承了HttpJspBase，而HttpJspBase则直接继承了HttpServlet类

JAP三种语法

JSP头部的Page指令

  • 可以修改jsp页面中一些重要属性或行为
  1. language jsp翻译后是什么语言文件，只支持Java
  2. contentType jsp返回的数据类型，也是源码中response.setContentType()参数值
  3. pageEncoding 当前jsp页面文件本身字符集
  4. import 用于导包导类

以下两个属性给out输出流使用

  1. autoFlush 设置当out输出流缓冲区满后是否自动刷新缓冲区，默认true，满后不刷新会报错

  2. buffer 设置out缓冲区的大小，默认8kb

  3. errorPage 设置当jsp页面运行时出错，自动跳转去的错误页面路径，一般以/开头

  4. isErrorPage 设置当前jsp页面是否是错误信息页面，默认false，若是true可以获取异常信息

  5. session 设置访问当前jsp页面是否会创建HttpSession对象，默认true

  6. extends 设置jsp翻译出来的Java类默认继承谁

JSP常用脚本

声明脚本（极少使用）

格式：<%! 声明Java代码 %>

作用：给jsp翻译出来的Java类定义属性、静态代码块、内部类等

表达式脚本（常用）

格式：<%=表达式%>

作用：在jsp页面上输出数据

特点·：

  1. 表达式脚本都会被翻译到jspService方法中

  2. 表达式脚本都会被翻译成out.print()输出到页面上

  3. 由于表达式脚本翻译的内容都在_jspService()方法中，所以该方法中的对象可以直接使用

  4. 表达式脚本中的表达式不能以分号结尾

代码脚本

格式：<% java语句 %>

作用：在jsp页面中编写自己需要的功能

特点：

  1. 翻译后都在_jspService()方法中
  2. 由于翻译到_ jspService()方法中，所以在_ jspService()方法中的现有对象都可以使用
  3. 代码脚本还可以由多个代码脚本块组合完成一个完整的Java语句
  4. 还可以和表达式脚本组合使用在页面上输出数据

JSP三种注释

html注释

<!-- 这是html注释 -->

会被翻译到java语言代码中，在_jspService()方法中以out.write输出到1客户端

Java注释

单行：// 单行注释

多行：/* 多行注释 */

会被翻译到Java源代码中

JSP注释

<%-- 这是jsp注释 --%>

可以注释掉jap页面中所有代码

JSP九大内置对象

  • JSP内置对象指Tomcat翻译jsp页面为Servlet源代码后，内部提供的九个对象
  1. request 请求对象
  2. response 响应对象
  3. pageContext jsp上下文对象
  4. session 会话对象
  5. application ServletContext对象
  6. config ServletConfig对象
  7. out jsp输出流对象
  8. page 指向当前jsp的对象
  9. exception 异常对象

四大域对象

根据其范围从小到大排列：（使用优先级也从小到大）

  • page域 ==》pageContext（pageContextlmpl类）：只能在当前页面有效。
  • request域 ==》 request（HttpServletRequest类）：只在一次请求或请求域中有效。
  • session域 ==》 session（HttpSession类）：一次会话（一次或多次请求和响应）过程中有效。
  • application域 ==》 application（ServletContext类）：整个项目过程中都有效

域对象是可以像map一样存储数据的对象，功能一样，对数据的存取范围不同

设定域对象的值

使用的方法都是setAttribute(“key”, “value”)方法，参数以键值对的方式存值

普通方式取值

以page域对象为例，其格式为：page: ${pageScope.pageKey}

pageKey为page域对象对应的key，使用这种方式就能取出域对象中的值了。

简写方式取值

以page域对象为例，格式为：page: ${pageKey}，将pageScope简化了

page域对象中的key和request域对象中的key可能一样，如果key重复了，会按照从小到大逐级查找。

out输出和response输出的区别

  • 因为jsp翻译后底层都用out进行输出，一般情况下同一使用out进行输出，避免打乱顺序

  • out.write()用于专门输出字符串

  • out.print()可以输出任意数据（都转换为字符串后调用write输出）

JSP常用标签

JSP静态包含（常用）

<%@ include file="/" %>表示http://ip:port/工程路径/，映射到代码的web目录

file属性是指定要包含的jsp页面的路径

特点：

  1. 不会翻译被包含的jsp页面
  2. 其实是把被包含的jsp页面的代码拷贝到包含位置进行输出

JSP动态包含

<jsp:include page="/工程名/XXX.jsp"></jsp:include>

page属性是指定要包含的jsp页面的路径

功能像静态包含一样

特点：会把包含的jsp页面也翻译成Java代码

JSP请求转发

<jap:forward page=""></jsp:forward>

EL表达式

  1. EL表达式全称：Expression Language，即表达式语言
  2. EL表达式作用：代替JSP页面中表达式脚本进行数据的输出(只能获取数据，不能设置数据)
  3. EL表达式的格式是：${表达式} ，注：EL表达式写在jsp页面中，表达式一般是域对象的key

输出格式：${表达式}

  • 在输出null时输出的是空串，jsp输出的是null字符串

EL表达式搜索域数据的顺序

  • EL表达式主要是输出域对象中的数据，当四个域对象都有同一个key的值时，EL表达式会按照四个域对象的范围从小到大进行搜索，找到就输出，与四个域对象声明的先后顺序无关

EL表达式输出Java类的属性

EL表达式的运算

  • 语法：${运算表达式}

关系运算

关系运算符 说 明 范 例 结果
== 或 eq 等于 $ {5==5} 或 $ {5eq5} true
!= 或 ne 不等于 $ {5!=5}或 $ {5ne5} false
< 或 lt 小于 $ {3<5}或 $ {3lt5} true
> 或 gt 大于 $ {2>10}或 $ {2gt10} false
<= 或 le 小于等于 $ {5<=12}或 $ {5le12} true
>= 或 ge 大于等于 $ {3>=5}或 $ {3ge5} false

逻辑运算

逻辑运算符 说 明 范 例 结果
&& 或 and 与运算 $ {12 == 12 && 12 < 11} 或 $ {12 == 12 and 12 < 11} false
||或 or 或运算 $ {12 == 12 || 12 < 11} 或 $ {12 == 12 or 12 < 11} true
! 或 not 取反运算 $ {true}或 $ {not true} false

算数运算

算数运算符 说明 范例 结果
+ 加法 $ {12+18} 30
- 减法 $ {18-8} 10
* 乘法 $ {12*12} 144
/ 或 div 除法 $ {144/12}或 $ {144div12} 12
% 或 mod 取模 $ {144%10}或 $ {144mod10} 4

empty运算

empty运算可以判断一个数据是否为空，若为空，输出true，不为空，输出false，${ empty key }
以下几种情况为空(在原本的key之前加empty关键字)：

  1. 值为null、空串
  2. 值为Object类型的数组且长度为0 (注：其他类型的长度为0的数组值为非空)
  3. List、Map集合元素个数为0

三元运算

  • 表达式 1？表达式 2：表达式 3
    表达式1为真返回表达式2的值，表达式1为假返回表达式3的值

“.”点运算和“[ ]”中括号运算

  • 点运算可以输出某个对象的某个属性的值(getXxx或isXxx方法返回的值)
  • 中括号运算可以输出有序集合中某个元素的值

注：中括号运算可以输出Map集合中key里含有特殊字符的key的值

EL表达式的11个隐含对象

EL表达式中的11个隐含对象是EL表达式自己定义的，可以直接使用

变量 类型 作用
pageContext PageContextImpl 它可以获取jsp中的九大内置对象,不是11个隐含对象
pageScope Map<String,Object> 它可以获取pageContext域中的数据
requestScope Map<String,Object> 它可以获取Request域中的数据
sessionScope Map<String,Object> 它可以获取Session域中的数据
applicationScope Map<String,Object> 它可以获取ServletContext域中的数据

header Map<String,String> 它可以获取请求头的信息
headerValues Map<String,String[]> 它可以获取请求头的值息，它可以获取多个值的情况
cookie Map<String,Cookie> 它可以获取当前请求的Cookie信息
initParam Map<String,String> 它可以获取在web.xml中配置的<context-param>参数

JSTL标签库

  • Java Standard Tag Lib（Java标准的标签库）。

  • JSTL标签库通常结合EL表达式一起使用，目的是让JSP中的java代码消失。

  • 标签是写在JSP当中的，但实际上最终还是要执行对应的java程序（java程序在jar包当中）

JSTL中定义了五个标签库

  • 核心标签库：主要用于完成基本的逻辑运算(重点)
    • 导入url：http://java.sun.com/jsp/jstl/core
  • 格式化标签库：主要用于完成日期、数字的格式化显示
  • EL函数标签库：定义了若干个EL函数
  • SQL操作标签库：完成SQL操作(已经不使用，由java完成)
  • XML操作标签库：完成XML操作(已经不使用，由java完成)
写JSTL之前需要导入两个jar包
  • jstl.jar
  • standard.jar
taglib指令在核心标签库中的使用
  • 使用JSTL核心标签库，需要在页面中通过taglib指令，将标签库导入
  • <%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c" %>

核心标签库常用的标签

标签名称 功能分类 分类 作用
<c:set/> 数据存储 核心标签库 存储数据
<c:if> 流程控制 核心标签库 用于判断
<c:choose> ,<c:when>,<c:otherwise> 流程控制 核心标签库 用于多个条件判断
<c:forEach> 迭代操作 核心标签库 用于循环遍历

<c:set/>

  • 用于往域中保存数据

<c:if />

  • 做if判断（只能做if判断，不可以if-else）

<c:choose> <c:when> <c:otherwise>

  • 可以做多路判断（与switch-case-default类似）

<c:forEach >

  • 遍历普通数据
  • 遍历Object数组
  • 遍历Map
  • 遍历List
  • forEach标签组合使用去遍历
