数组

  • 线性表: 线性表就是数据排成像一条线一样的结构.每个现行表上的数据最多只有前和后两个方向.常见的线性表结构：数组，链表、队列、栈等。

概述

  • 数组是由一组类型相同的元素（值或变量）组成的数据结构，每个元素有至少一个索引或键来标识

  • 因为数组内的元素是存储在连续内存空间中的，所以数组中元素的地址，可以通过其索引计算出来

  • 知道了数组的数据起始地址 $BaseAddress$，就可以由公式 $BaseAddress + i * size$ 计算出索引 $i$ 元素的地址

    • $i$ 即索引，在 Java、C 等语言都是从 0 开始
    • $size$ 是每个元素占用字节，例如 $int$ 占 $4$，$double$ 占 $8$

Java实现

Array数组

  • 实现：int[], String[], 等基本类型和对象数组。
  • 常用API：
    • array.length：获取数组长度。
    • array[index]：访问或修改数组元素。

    • replace(char oldChar, char newChar)：替换字符串中的元素

空间占用

Java 中数组结构为

  • 8 字节 markword
  • 4 字节 class 指针（压缩 class 指针的情况）
  • 4 字节 数组大小（决定了数组最大容量是 $2^{32}$）
  • 数组元素 + 对齐字节（java 中所有对象大小都是 8 字节的整数倍[^12]，不足的要用对齐字节补足）

操作性能

随机访问性能

即根据索引查找元素，时间复杂度是 $O(1)$

根据值查询性能

时间复杂度是 $O(n)$

插入或删除性能

头部位置，时间复杂度是 $O(n)$，涉及到数据移动

中间位置，时间复杂度是 $O(n)$，涉及到数据移动

尾部位置，时间复杂度是 $O(1)$（扩容并非每次都发生，均摊来说）

动态数组

  • Java自带的数组不能插入/删除元素，并且创建时已经固定容量，被称为静态数组
  • 能够插入/删除元素，大小可以根据需要变化，即动态数据，Java内已有ArrayList实现

缓存与局部性原理

  • 先遍历列和先遍历行在效率上有区别

  • ij 方法

    ji 方法

  • 先遍历列（ij）比先遍历行（ji）效率高很多

    • CPU计算从内存中取得数据，将结果写回内存，但CPU读取速度跟不上运算速度，此时就需要缓存
    • 一次放入缓存的数据会有64字节，即缓存行，该数据会与其邻近数据组成缓存行写入，即空间局部性
    • 缓存是有限的，当新数据来了后，一些旧的缓存行数据就会被覆盖
    • 如果不能充分利用缓存的数据，就会造成效率低下

  • 先遍历行（ji）：

    • 第一次内循环要读入 $[0,0]$ 这条数据，由于局部性原理，读入 $[0,0]$ 的同时也读入了 $[0,1] … [0,13]$，如图所示

    • 第二次内循环读取是 $[1,0]$ 这条数据，缓存中没有，于是再读入了下图的数

    • 这显然是一种浪费，因为 $[0,1] … [0,13]$ 包括 $[1,1] … [1,13]$ 这些数据虽然读入了缓存，却没有及时用上，而缓存的大小是有限的，等执行到第九次内循环时，缓存的第一行数据已经被新的数据 $[8,0] … [8,13]$ 覆盖掉了，以后如果再想读，比如 $[0,1]$，又得到内存去读了

  • 同理：

    • I/O 读写时同样可以体现局部性原理
    • 数组可以充分利用局部性原理，但链表不行，因为链表的元素并非相邻存储

链表

介绍

  • 链表（Linked List）是数据元素的线性集合，其每个元素都指向下一个元素，元素存储上并不连续

  • 分类

    • 单链表，每个元素只知道其下一个元素是谁

    • 双向链表，每个元素知道其上一个元素和下一个元素

    • 循环链表，通常的链表尾节点 tail 指向的都是 null，而循环链表的 tail 指向的是头节点 head

  • 链表内还有特殊节点：哨兵节点（Sentinel Node），也称为哑元节点（Dummy Node），其值域不存储任何链表数据，其主要作用是简化链表插入和删除操作的条件判断。拥有哨兵节点的链表称为带头链表

Java实现

LinkedList

  • 实现：java.util.LinkedList。
  • 常用API：
    • add(E e)：添加元素到链表末尾。
    • add(int index, E element)：在指定位置添加元素。
    • remove(int index)：移除指定位置的元素。
    • get(int index)：获取指定位置的元素。

性能

随机访问性能

根据 index 查找，因为不是连续存储的，只能从头节点开始查找，时间复杂度为 $O(n)$

插入/删除性能

  • 整体时间复杂度 = 查找时间 + 插入时间，若已知插入节点又有不同

  • 起始位置：$O(1)$

  • 结束位置：如果已知 tail 尾节点是 $O(1)$，不知道 tail 尾节点是 $O(n)$

  • 中间位置：根据 index 查找时间 + $O(1)$

双向链表（带哨兵）

  • 双链表的哨兵有头尾两个，空链表时，头尾相连
  • 双向链表优势在于尾节点已知，在尾节点查找方面有便利

区分哨兵节点与头节点：

  • 哨兵节点位于链表的最前端，其next指针指向链表的第一个实际节点（也就是头节点）
  • 头节点是链表中第一个存储有效数据的节点。在没有哨兵节点的链表中，头节点同时也是链表的起始点

环状链表（带哨兵）

环状链表的哨兵既作为头，也作为尾，分为单向和双向

递归定义

  • 递归是一种解决计算问题的方法，其中解决方案取决于同一类问题的更小子集

  • 说明：

    1. 自己调用自己，如果说每个函数对应着一种解决方案，自己调用自己意味着解决方案是一样的（有规律的）
    2. 每次调用，函数处理的数据会较上次缩减（子集），而且最后会缩减至无需继续递归
    3. 内层函数调用（子集处理）完成，外层函数才能算调用完成

  • 优点：代码的表达力很强，写起来简洁。

    缺点：空间复杂度高、有堆栈溢出风险、存在重复计算、过多的函数调用会耗时较多等问题

思路

思路(此处为代码随想录递归三部曲)

  1. 确定能否使用递归求解

    • 一个问题只要同时满足以下3个条件，就可以用递归来解决：
      • 问题的解可以分解为几个子问题的解。何为子问题？就是数据规模更小的问题。
      • 问题与子问题，除了数据规模不同，求解思路完全一样
      • 存在递归终止条件

  2. 确定递归函数的参数和返回值： 确定哪些参数是递归的过程中需要处理的，那么就在递归函数里加上这个参数， 并且还要明确每次递归的返回值是什么进而确定递归函数的返回类型。

  3. 确定终止条件： 写完了递归算法, 运行的时候，经常会遇到栈溢出的错误，就是没写终止条件或者终止条件写的不对，操作系统也是用一个栈的结构来保存每一层递归的信息，如果递归没有终止，操作系统的内存栈必然就会溢出。

  4. 确定单层递归的逻辑： 确定每一层递归需要处理的信息。在这里也就会重复调用自己来实现递归的过程。

  • 深入到最里层叫做递
  • 从最里层出来叫做归
  • 在递的过程中，外层函数内的局部变量（以及方法参数）并未消失，归的时候还可以用到

冒泡排序

  • 基本思想是：对待排序序列遍历，依次比较相邻元素的值，若发现前数大于后数则交换，使值较大的元素逐渐从前移向后部。
  • 每一轮排序可以确定一个数的位置，一共进行n-1次

排序（冒泡排序，选择排序，插入排序，归并排序，快速排序，计数排序，基数排序） - VisuAlgo

  • 已排序区域 ( i , j ]
  • 未排序区域 [ 0 , i ]，i赋值给x则表示 [ 0 , x ]区域未排序，x不用自减，最后完全排序后x = 0，提前结束递归

插入排序

  • 通过构建有序序列（可以将第一个元素列为已排序序列），从前往后提取未排序数据，在已排序序列中从后向前扫描，找到相应位置并插入

变体问题

兔子问题

  • 有一对小兔子，从出生后第3个月起每个月都生一对兔子。小兔子长到第3个月后每个月又生一对兔子。按此规律，假设没有兔子死亡，第一个月有一对刚出生的小兔子，问第n个月有多少对兔子？

分析

兔子问题如何与斐波那契联系起来呢？设第 n 个月兔子数为 $f(n)$

  • $f(n)$ = 上个月兔子数 + 新生的小兔子数
  • 而【新生的小兔子数】实际就是【上个月成熟的兔子数】
  • 因为需要一个月兔子就成熟，所以【上个月成熟的兔子数】也就是【上上个月的兔子数】
  • 上个月兔子数，即 $f(n-1)$
  • 上上个月的兔子数，即 $f(n-2)$

因此本质还是斐波那契数列，只是从其第一项开始

递归优化-记忆法

  • 该算法的时间复杂度为$\Theta(1.618^n)$

  • 原来的代码包含许多重复计算

  • 如果能够记录这些重复计算就能减少运行时间

Memoization 记忆法（也称备忘录）是一种优化技术，通过存储函数调用结果（通常比较昂贵），当再次出现相同的输入（子问题）时，就能实现加速效果，改进后的代码

  • 改进后的时间复杂度为 $O(n)

注意

  1. 记忆法是动态规划的一种情况，强调的是自顶向下的解决
  2. 记忆法的本质是空间换时间

爆栈问题

  • 每次方法调用是需要消耗一定的栈内存的，这些内存用来存储方法参数、方法内局部变量、返回地址等等
  • 方法调用占用的内存需要等到方法结束时才会释放
  • 如果栈内存占满，则发生爆栈

尾调用

如果函数的最后一步是调用一个函数，那么称为尾调用

  • 某些编辑器可以对尾调用进行优化，成一种平行的调用关系，这是外层函数可以在调用内层前结束，从而释放内存

汉诺塔问题

Tower of Hanoi，是一个源于印度古老传说：大梵天创建世界时做了三根金刚石柱，在一根柱子从下往上按大小顺序摞着 64 片黄金圆盘，大梵天命令婆罗门把圆盘重新摆放在另一根柱子上，并且规定

  • 一次只能移动一个圆盘
  • 小圆盘上不能放大圆盘

分析

  • 要把n个圆盘从柱一移到柱三，必须把前n - 1个圆盘移到柱二

杨辉三角

  • 杨辉三角是一个三角形阵列，其中每个数字是它正上方两个数字之和。
  • 行 $i$，列 $j$，那么 $[i][j]$ 的取值应为 $[i-1][j-1] + [i-1][j]$
  • 当 $j=0$ 或 $i=j$ 时，$[i][j]$ 取值为 $1$

题解

  • 但该方法时间复杂度为指数级

优化

  • 是 multiple recursion，因此很多递归调用是重复的
  • 以下对空间复杂度加以优化(简化为一维数组)

主定理求时间复杂度

若有递归式
$$
T(n) = aT(\frac{n}{b}) + f(n)
$$
其中

  • $T(n)$ 是问题的运行时间，$n$ 是数据规模
  • $a$ 是子问题个数
  • $T(\frac{n}{b})$ 是子问题运行时间，每个子问题被拆成原问题数据规模的 $\frac{n}{b}$
  • $f(n)$ 是除递归外执行的计算

令 $x = \log_{b}{a}$，即 $x = \log_{子问题缩小倍数}{子问题个数}$

那么
$$
T(n) =
\begin{cases}
\Theta(n^x) & f(n) = O(n^c) 并且 c \lt x\
\Theta(n^x\log{n}) & f(n) = \Theta(n^x)\
\Theta(n^c) & f(n) = \Omega(n^c) 并且 c \gt x
\end{cases}
$$

展开求解

像下面的递归式，都不能用主定理求解

例1 - 递归求和

$T(n) = T(n-1) + c$，$T(1) = c$

下面为展开过程

$T(n) = T(n-2) + c + c$

$T(n) = T(n-3) + c + c + c$

…

$T(n) = T(n-(n-1)) + (n-1)c$

  • 其中 $T(n-(n-1))$ 即 $T(1)$
  • 带入求得 $T(n) = c + (n-1)c = nc$

时间复杂度为 $O(n)$

例2 - 递归冒泡排序

$T(n) = T(n-1) + n$，$T(1) = c$

下面为展开过程

$T(n) = T(n-2) + (n-1) + n$

$T(n) = T(n-3) + (n-2) + (n-1) + n$

…

$T(n) = T(1) + 2 + … + n = T(1) + (n-1)\frac{2+n}{2} = c + \frac{n^2}{2} + \frac{n}{2} -1$

时间复杂度 $O(n^2)$

注：

  • 等差数列求和为 $个数*\frac{\vert首项-末项\vert}{2}$

例3 - 递归快排

快速排序分区没分好的极端情况

$T(n) = T(n-1) + T(1) + n$，$T(1) = c$

$T(n) = T(n-1) + c + n$

下面为展开过程

$T(n) = T(n-2) + c + (n-1) + c + n$

$T(n) = T(n-3) + c + (n-2) + c + (n-1) + c + n$

…

$T(n) = T(n-(n-1)) + (n-1)c + 2+…+n = \frac{n^2}{2} + \frac{2cn+n}{2} -1$

时间复杂度 $O(n^2)$
