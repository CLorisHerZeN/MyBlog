哈希表

发表于2024-10-25|更新于2024-10-28
|字数总计:1.6k|阅读时长:7分钟|阅读量:
  • 哈希表通过键（key）来存储和访问值（value）。每个键在哈希表中都是唯一的
  • 给每份数据分配一个编号，放入表格（数组），提供了平均时间复杂度为O(1)的查找、插入和删除操作。
  • 建立编号与表格索引的关系，将来就可以通过编号快速查找数据
    • 理想情况：编号唯一，数组能容纳所有数据
    • 现实是无法为容纳所有数据建立过大的数组，编号有可能重复
    • 解决：
      • 有限长度的数组，以拉链的方式存储数据（但是会造成性能上的降低）
      • 允许编号适当重复，通过数据自身来区分

生成 hashCode

  • 哈希表使用哈希函数将键映射到表中的一个位置（称为哈希桶或哈希槽）。哈希函数的设计目标是尽可能均匀地将键分布到哈希表的各个位置，以减少冲突。
  • hash 算法（摘要算法，散列算法）是将任意对象，分配一个编号的过程，其中编号是一个有限范围内的数字（如 int 范围内）
  • 最常用的hash算法有MD5和SHA系列算法，在计算hash值时都考虑了原始数据的每一个字节，一旦改动，hash值就会有明显不同，因此被广泛用于数据完整性检验和加密等方面

实现

使用Object哈希码

Object.hashCode

  • Object 的 hashCode 方法默认是生成随机数作为 hash 值（会缓存在对象头当中）
  • 缺点是包含相同值的不同对象，他们的 hashCode 不一样，不能够用 hash 值来反映对象的值特征，因此诸多子类都会重写 hashCode 方法

生成字符串哈希码

  • 原则：值相同的字符串生成相同的hash码，尽量让值不同的字符串生成不同的hash码

String.hashCode

  • 经验表明如果每次乘的是较大质数，可以有更好地降低 hash 冲突，因此改【乘 10】为【乘 31】
  • 【乘 31】可以等价为【乘 32 - hash】，进一步可以转为更高效地【左移5位 - hash】
