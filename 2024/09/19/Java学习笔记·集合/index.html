集合的框架体系

  • Java 提供了一系列集合容器，以方便程序员动态保存元素。并提供了一系列方便的操作对象的方法。

  • Java 集合主要分为两组：单列集合（Collection）、双列集合（Map）

单列集合接口Collection

  • Collection接口有两个重要的子接口List（有序集合）、Set（无序集合），它们实现的子类都是单列集合（存放单列数据）
  • Collection实现子类可以存放多个元素（可以是Object）
  • 有些实现子类可以存放重复的元素，有些不能；有些是有序的，有些不是
  • Collection接口没有直接的实现子类，是通过它的子接口Set和List来实现的

常用方法

  • add：添加单个元素

  • remove：删除单个元素，若删除指定对象返回是否删除成功的布尔值，按索引删除返回对象本身

  • contains()：检查元素是否存在

  • size()：获取元素个数

  • isEmpty()：判断是否为空

  • clear()：清空

  • addAll()：添加多个元素

  • containsAll：检查多个元素是否存在

  • removeAll：删除多个元素

迭代器 Iterator

  • Collection 继承的 Iterable 接口中，提供了 iterator() 方法，会返回一个迭代器对象。
  • 迭代器用于遍历Collection集合中的元素，Iterator 对象仅用于遍历集合，本身不存放元素
  • 所有实现了Collection接口的子类都有 iterator() 方法

遍历元素：

方法一：迭代器遍历

  • boolean hasNext()：该方法判断是否有下一个元素。
  • T next()：该方法会将指针下移，然后返回下移后的位置上的元素

  1. 获取迭代器

  2. 判断有无下一元素

  3. 将迭代器后移，并返回那个后移位置上的元素

    while 循环结束后，指针指向最后元素的位置。再次 next() 会报错。

    如果需要再次遍历，需要重置迭代器。

方法二：for循环增强

  • 就是简化版的iterator，底层也仍然是迭代器，只能用于遍历集合或数组
  • 快捷键：I（大写i）

有序集合接口 List

  • List 是 Collection 接口的子类接口
  • List 是有序（添加顺序和取出顺序一致）的，可重复的
  • List 中的每个元素都有其对应的整数型顺序索引（从 0 开始编号），即支持索引
  • 实现List接口的常用实现类有：ArrayList、LinkList和Vector

常用方法

  • add(int, obj)：在 int 位置插入 obj 元素。返回 true

    add(obj)：在末尾插入 obj。返回 true

  • addAll(int, collection)：在 int 位置插入 collection 中的所有元素

  • get(int)：返回 int 位置的元素

  • indexOf(obj)：返回 obj 首次出现时的位置

  • lastIndexOf(obj)：返回 obj 最后一次出现时的位置

  • remove(int)：移除 int 位置的元素，并返回那个被移除的元素

  • set(int, obj)：设置 int 位置的元素为 obj。相当于替换。返回那个被替换元素的下标

  • subList(int1, int2)：返回 [int1, int2) 范围的元素构成的子集合

三种遍历方式

  1. 迭代器

  2. 增强for

  3. 普通for

  • 其它List接口的实现类遍历方法也是一致的

可变数组ArrayList

  • 可以放任何值，可加入多个null
  • 由数组来实现存储
  • 基本等同于Vector，但线程不安全（执行效率高）

底层结构与源码分析

  • ArrayList 中维护了一个 Object 类型的数组 elementData。该数组就是用来存放元素的数组
  1. 创建 ArrayList 对象时，如果使用无参构造器，则 elementData[] 初始容量为 0，第一次添加，扩容为10，如需再次扩容，则扩容为1.5倍
  • 执行add()方法时
  • 扩容
  1. 如果使用指定大小构造器，则初始容量为指定大小。如需再次扩容，则扩容为1.5倍

可变数组Vector

  • 定义说明：Vector 是 List 的实现子类。
  • 底层也是一个对象数组protected Object[] elementData;
  • 在开发中，需要考虑线程安全时，建议使用 Vector

Vector与ArrayList的比较

底层结构 版本 线程安全（同步）效率 扩容倍数
ArrayList 可变数组 jdk1.2 不安全，效率高 有参1.5倍，无参：第一次10，第二次1.5倍
Vector 可变数组Object[] jdk1.0 安全，效率不高 无参10，满后2倍；有参直接2倍

底层结构与源码分析

  1. 底层维护了一个 Object 类型的数组 elementData。用以存放元素

  2. 使用无参构造器创建对象时，默认大小是 10

    使用有参构造器的场合，默认是那个指定大小（initialCapaticy）

  3. 调用vector.add(i)

  4. 扩容的场合，容量变成 2 倍

    使用有参构造器改变了 capacityIncrement 的场合，增量是那个指定数值

链表LinkedList

  • 底层实现了双向链表和双端队列特点
  • 可以添加任意元素。包括null，可重复
  • 线程不安全，没有实现同步
  • 添加和删除不是通过数组完成，效率更高

底层结构与源码分析

  1. LinkedList 只有默认构造器和一个拷贝构造器

  2. LinkedList 底层维护了一个 双向链表

    两个属性 first、last 分别指向 首节点 和 尾节点

    每个节点（Node 对象），里面又维护了 prev、next、item 属性。

    其中通过 prev 指向前一个节点，通过 next 指向后一个节点。最终实现双向链表。

  3. 调用add方法

  4. LinkedList 不需要扩容。其增删元素时只要改变节点的指向即可。

    也因此，其添加、删除元素效率比数组更高

  5. 调用linkedList.remove();默认删除第一个节点

ArrayList 和 LinkedList 的比较

底层结构 增删效率 改查效率
ArrayList 可变数组 低（数组扩容） 高
LinkedList 双向链表 高（链表追加） 低
  • 如果改查操作多，选择 ArrayList。一般来说，在程序中，80% - 90% 都是查询。大部分情况下，选择 ArrayList。
  • 如果增删操作多，选择 LinkedList

无序集合接口Set

共同特性

  • 无序，添加和取出顺序不一样，没有索引
  • 不允许重复元素，最多只能包含一个null
  • 常用子接口为HashSet与TreeSet

常用接口

Set也是Collection的子接口，常用方法与Collection一致

遍历方式

同Connection接口

  1. 可以使用迭代器
  2. 增强for
  3. 不能使用索引的方式获取

HashSet

  • 实现了Set接口，底层是HashMap（数组+链表+红黑树）
  • 添加和取出顺序不一样，但取出的顺序是固定的（底层是数组加链表）

底层结构与源码分析

  1. 第一次添加元素时，table数组扩容到16，临界值（threshold）是16*加载因子（loadFactor）0.75 = 12
  2. 添加元素时，先得到哈希值，再转成索引值
  3. 如果数组达到临界值就会扩容到16*2 = 32，新临界值是32 * 0.75 = 24，以此类推
  4. 临界值指只要每条链表挂载的对象总数超过就扩容，不是指table占据超过12
  5. 到数据存储表table，查看该索引是否存放元素，若无，直接加入
  6. 若有，调用equals方法比较，若相同，则放弃添加，若不同，则添加到链表最后
  7. Java8中，若一条链表的元素个数到达TREEIFY_THRESHOLD(默认是8)，并且table大小大于等于MIN_TREEIFY_CAPACITY(默认64)，就会进行树化（红黑树）

  2. 调用add方法

  3. 调用put方法，该方法会执行hash(key) 得到key 对应的hash 值(不等于hash code)

  4. 执行putVal

LinkedHashSet

  • LinkedHashSet 是 HashSet 的子类
  • LinkedHashSet 底层是一个 LinkedHashMap，维护了一个数组 + 双向链表。有其父必有其子
  • LinkedHashSet 根据元素的 hashCode 值来决定元素的存储位置。同时，使用链表维护元素的次序。这使得元素看起来是以插入顺序保存的，并得以按照放入顺序取出

底层结构与源码分析

  • 在LinkedHashSet中维护一个LinkedHashMap（是HashMap的子类）

  • 底层结构是数组table和双向链表（有head和tail）

  • 双向链表每个节点有before和after属性形成双向链表

    tail.next = newElement;

    newElement.pre = tail;

    tail = newElement;

  • 添加元素时操作同HashSet，这样遍历在LinkedHashSet时能够保证插入顺序与遍历顺序一致

双列集合接口Map

Map接口实现类的特点

  • Map接口的实现子类是双列集合，存放的是key和value（存放双列数据）
  • Map接口的常用实现类：HashMap,HashTable,Proprties

适用于 JDK 8 的环境

  • Map 与 Collection 并列存在，用于保存具有映射关系的数据：key - value（双列元素）
  • Map 的 key 和 value 可以是任何类型的引用数据类型，会封装到HashMap$Node对象中
  • Map 的 key 不允许重复，value 可以重复。key 和 value 存在单一对应关系。通过特定的 key 一定能找到指定的 value。当有相同的key，等价为替换
  • Map 的 key和value都可以为null，key为null只能有一个，value为null可有多个
  • 常用String类作为Map的key
  • 一组 k - v 会被封装到一个 Entry 对象中。Entry 是一个内部接口。Map 的实现子类中都包含一个实现这个接口的内部类。

Map接口和常用方法

Map接口的特点

  1. 键值对封装:
    • 在Map接口中，每个键值对都被封装在一个实现了Map.Entry接口的HashMapNode对象中。即EntrySet<Map.Entry<K, V>>
    • Map.Entry接口定义了键值对的基本操作，如获取键（getKey()）和获取值（getValue()，从而轻松地访问和修改Map中的键值对
  2. EntrySet集合:
    • Map接口提供了一个entrySet()方法，它可以返回一个集合，这个集合包含了Map中所有的键值对。
    • 这个集合被称为EntrySet，它存放的是Map.Entry类型的对象，每个对象的两个引用各自指向键和值。
  3. 键值对的存储:
    • 在实现Map接口的类（如HashMap）中，每个键值对实际上是通过一个实现了Map.Entry接口的类（如HashMap中的Node类）来存储的。
    • 这意味着虽然EntrySet中定义的类型是Map.Entry，但实际上存放的是具体的实现类（如HashMap中的Node类）的实例。
  • Map接口提供了一种抽象的方式，允许存储和操作键值对。通过实现Map.Entry接口的类，你可以将键值对存储在各种数据结构中，如链表、红黑树等。而EntrySet集合则提供了一种方便的方式来遍历和操作这些键值对。
  • Map接口定义了键值对集合的框架和操作，而Map.Entry接口则定义了键值对的具体表示和行为。

常用方法

  • put()：添加。已存在的场合，实行替换。（key 不替换，value 替换）

  • remove()：根据键删除映射关系

  • get()：根据键获取值

  • size()：元素个数

  • isEmpty()：判断个数是否为 0

  • clear()：清空

  • containsKey()：查找键是否存在

  • Set<K> keySet()：获取所有 键 构成的集合

    Set<Map.Entry<K,V>> entrySet()：获取所有 Entry 构成的集合

    Collection<V> values()：获取所有 值 构成的集合

Map接口遍历方式

  • 第一组：取出所有的key，通过key取出对应value

    • 法一：增强for

    • 法二：迭代器

  • 第二组：把所有的value取出，可以使用所有Collection的遍历方法

  • 第三组：通过EntrySet 来获取k-v

散列表HashMap

  • HashMap 是 Map 接口使用频率最高的实现类。
  • HashMap以key-value对的方式来存储数据（HashMap$Node类型），一组数据会被封装到一个 Node 对象中。
  • key不能重复，但value可以，允许null键与null值
  • 添加相同key，则覆盖原来的k-v
  • HashMap 没有实现同步（没有 synchronized），是线程不安全的

底层机制及源码解析

  • JDK 7 前，HashMap 底层是 数组 + 链表。JDK 8 后，底层是 数组 + 链表 + 红黑树。HashMap 不保证映射的顺序
  • 扩容机制与HashSet完全一致

  1. HashMap 底层维护了 Node 类型的数组 table。默认为 null

  2. 创建对象时，默认构造器将加载因子（loadfactor）初始化为 0.75。

    也能指定那些初始容量和加载因子。

    默认构造器第一次添加元素的场合，table 扩容为 16，临界值为 16 * 0.75 = 12。

  3. 添加时容量不够的场合，需要扩容。

    默认构造器第一次添加元素的场合，table 扩容为 16，临界值为 16 * 0.75 = 12。

    扩容的场合，容量变为 2 倍。临界值相应变化。

    临界值不会超过那个指定的 MAXIMUM_CAPACITY（1 << 30），否则变成 Integer.MAX_VALUE。

    JDK 8 中，如果一条链表的元素个数超过 TREEIFY_THRESHOLD（默认是 8），并且 table 的大小 >= MIN_TREEIFY_CAPACITY（默认 64），会进行树化。

    剪枝：红黑树的元素减少到一定程度，会被重新转化为 链表

    ​

  4. 添加 k - v 时，通过 key 的哈希值得到其在 table 的索引，判断索引位置是否被占用。

    未占用的场合，直接添加。

    占用的场合，判断其 key 是否相等。相等的场合，替换 value。否则，按照 树 或 链表 的方式处理。

散列表HashTable

  • Hashtable 和 HashMap 基本一致，但Hashtable 是线程安全的 。但也因为如此，Hashtable 的效率低下。
  • Hashtable不允许键和值为空

Hashtable 与 HashMap 的比较：

版本 线程安全（同步） 效率 是否允许 null值
Hashtable 1.0 安全 较低 不允许
HashMap 1.2 不安全 高 允许

底层机制

  1. Hashtable底层为数组HashTable$Entry[]，初始大小为11

  2. 临界值threshhold是 11 * 0.75 = 8。

  3. 扩容

  4. Hashtable 不会树化

Properties

  • Properties 继承自 Hashtable 并实现了 Map 接口。也使用键值对的方式保存数据
  • Properties 使用特点与 Hashtable 相似，k-v不能为空
  • Properties 还可以用于 xxx.properties 文件中，加载数据到 Properties 对象，进行读取和修改
  • xxx.properties 文件常作为配置文件(详见IO流)

  • 默认构造器创建TreeMap是无序的（也没有排序）

  • 传入比较器指定排序方式

集合选型规则

在开发中，选择什么集合实现类，主要取决于业务操作特点，然后根据集合实现类特性进行分析选择。

判断存储的类型（一组对象 [单列]，或一组键值对 [双列]）

  • 一组对象：Collection 接口
    • 允许重复：List
      • 增删多：LinkedList （双向链表）
      • 改查多：ArrayList （Object[] 数组）
    • 不允许重复：Set
      • 无序：HashSet （数组 + 链表 + 红黑树，底层是 HashMap）
      • 排序：TreeSet
      • 顺序一致：LinkedHashSet （数组 + 双向链表，底层是 LinkedHashMap）
  • 一组键值对：Map
    • 键无序：HashMap （数组 + 链表 + 红黑树 [ JDK 8 以后 ] ）
    • 键排序：TreeMap
    • 键顺序一致：LinkedHashMap （底层是 HashMap）
    • 读取文件：Properties

Collection工具类

  • Collections 工具类是一个操作 Set、List、Map 等集合的工具类
  • 其中提供了一系列静态方法，对集合元素进行 排序、查询和修改等操作

常用方法

排序：

  • Collections.reverse(List)：反转 List 中元素的排序
  • Collections.shuffle(List)：对 List 中元素进行随机排序
  • Collections.sort(List)：根据元素的自然顺序(首字母大小)对指定 List 集合元素升序排列
  • Collections.reverse(List, Comparator)：根据指定 Comparator 对 List 排序
  • Collections.swap(List, int, int)：将两处元素位置互换

查找、替换：

  • Collections.max(Collection)：根据元素的自然排序，返回集合中最大的元素

  • Collections.max(Collection, Comparator)：根据比较器，返回最大元素（如长度最大）

  • Collections.min(Collection)：根据元素的自然排序，返回最小元素

  • Collections.min(Collection, Comparator)：根据比较器，返回最小元素

  • Collections.frequency(Collection, Object)：返回集合中指定元素的出现次数（int）

  • Collections.copy(List dest, List list)：将 list的内容复制到 dest 中

    这个场合，要保证 dest 的大小不小于list。所以，可能需要先给 dest 赋值

  • Collections.replaceAll(List list, Object oldVal, Object newVal)：用 newVal 替换所有 oldVal 值
