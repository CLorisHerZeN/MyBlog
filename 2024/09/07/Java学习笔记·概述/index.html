Java学习笔记·概述

|字数总计:1.7k|阅读时长:5分钟|阅读量:

Java特性

  1. Java 语言是面向对象的（oop），将重点放在对象及其接口上

  2. Java 语言是健壮的。其强类型机制、异常处理、垃圾自动收集是健壮性的保证。

  3. Java 语言是解释型的：解释型语言编译后需要解释器才能运行。相对的，编译型语言可以被直接执行。

    Java 解释器能在任何移植了解释器的机器上直接执行 Java 字节码。

搭建开发环境

  • JDK = JRE + 开发工具集（例如Javac,java 编译工具等) —— 编写Java程序的程序员使用的软件
  • JRE = JVM + Java SE 标准类库（java 核心类库）—— 运行Java程序的用户使用的软件

JDK（Java Development Kit－ Java 开发工具包）

  • 供开发人员使用

JRE（Java Runtime Environment －Java 运行环境)）

  • 如果想要运行一个开发好的Java 程序，计算机中只需要安装JRE 即可

Java 核心机制-JVM(java virtual machine - Java 虚拟机)

  • 一个虚拟的计算机，负责执行指令，管理数据、内存、寄存器，包含在JDK 中
  • 不同平台有不同虚拟机
  • Java 虚拟机机制屏蔽了底层运行平台的差别，实现了“一次编译，到处运行”

环境变量path配置及其作用

  1. 环境变量的作用是为了在dos的任意目录，可以去使用java 和 javac命令
  2. 先配置 JAVA_HOME = 指向jdk安装的主目录
  3. 编辑path环境变量，增加 %JAVA_HOME%\bin

开发细节

运行原理

  • .java 称为源文件， .class称为字节码文件

注意事项及细节说明

  1. 源文件以 .java 为扩展名，源文件的基本组成部分是类（class），程序的所有内容都必须放置在类中

  2. Java 应用程序的执行入口是 main() 方法。其有固定的书写格式：

    public static void main(string[]args){…}

  3. Java 语言严格区分大小写。

  4. Java 方法由一条条语句构成，每个语句都以 ; 结束。

  5. 大括号 { } 是成对出现的，缺一不可。习惯先写 {} 再写代码

  6. 一个源文件中最多只有一个 public 类，其余类不限。

  7. 如果文件中包含 public 类，则文件名必须按该类命名。

  8. 也可以把 main 方法写在非 public 类中，然后运行指定非 public 类，这样入口方法是非 public 类的主方法。

  9. 在控制台按 tab 可以实现代码补齐。按方向键 ↑ 或 ↓ 可以调用历史代码。

代码规范

  1. 类、方法的注释要以 Javadoc 的方式来写（使用特定标签）

  2. 非 Javadoc 的注释，往往是给维护者看的，着重告诉读者为什么这样写，如何修改，注意什么问题等。

  3. 不要用 a b，这种名称命名变量，尽量写得清楚 int age = 10;

    另外，Java 源代码使用的是 Unicode 码，因此汉语也能作为标识符。但不推荐使用汉语做标识符。

  4. 使用 tab 键操作，使代码右移。使用 shift+tab 键，使代码左移。

  5. 运算符两边各加入空格。注意排版规范。

  6. 源文件使用 UTF-8 编码。

  7. 代码行宽度不要超过 80 个字符。超过时通过换行保持简洁。

  8. 代码编写次行风格及行尾风格

    次行风格：换行输入{ }，使其总在行头

    行尾风格：在一行的末尾输入 {，换行输入 }

  9. 一段代码完成一个小功能，尽量不要混合。这样更加灵活。

转义字符

注释

单行注释

  • 选中文字按 ctrl + /

多行注释

  • 选中文字按 ctrl + shift +/

文档注释

  • 文档注释的注释内容可被 JDK 中的 Javadoc 工具解析，生成一套以 HTML 形式体现的说明文档。（每句前不写星号也合法）

  • 抽取注释：javadoc -d 生成目录 -author -..（一系列标签） 文档名.java

  • 以下是一些常用的 Javadoc 标签及其示例：

    • @author - 指定类的作者。

    • @version - 指定类的版本号。

    • @param - 描述方法或构造函数的参数。

    • @return - 描述方法或构造函数的返回值。

    • @throws - 描述方法可能抛出的异常。

    • @see - 参考其他类、方法或字段。

    • @since - 指定一个方法或类从哪个版本开始存在。

    • @deprecated - 标记一个类、方法或字段为过时的。

    • @inheritDoc - 表示当前方法的文档应该与父类的方法文档相同。

DOS（了解）

原理

  • 控制台将指令发送至DOS系统
  • DOS系统接受、解析并执行指令，体现在Windows文件系统

相对路径与绝对路径

  • 相对路径：相对当前目录进行移动查找 ./ 表示从当前目录开始，../表示返回上一级
  • 绝对路径：从盘符出发d:\Program\JDK8\jre\bin

指令

  • 查看帮助：help 或 help cd 等

  • 查看目录内容：dir 查看当前目录 或 dir d:\Program\JDK8\bin 查看指定目录

  • 切换到其他盘：cd /D d: 从 C盘 切换至 D盘

  • 切换到当前盘的其他目录：cd d:\Program\JDK8\jre\bin

  • 返回上级目录：cd ..

  • 切换至根目录：cd \

  • 查看子集目录：tree 当前目录 或 tree d:/Program 指定目录

  • 清屏：cls

  • 退出：exit

  • 创建/删除目录：md 目录名 rd 目录名

  • 拷贝/删除文件：copy 文件名 目录 del 文件名

  • 移动文件：move 文件名 目录
